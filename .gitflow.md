# GitFlow - Estrutura de Branches

Este projeto segue o modelo GitFlow para gerenciamento de versões.

## Branches Principais

- **main**: Branch de produção. Contém código estável e testado.
- **develop**: Branch de desenvolvimento. Integra todas as features.

## Branches de Suporte

- **feature/***: Novas funcionalidades desenvolvidas a partir de `develop`
- **release/***: Preparação de novas versões para produção
- **hotfix/***: Correções urgentes aplicadas diretamente em `main`

## Estrutura Atual

```
main (v1.0.0)
  └── develop
```

## Como Trabalhar com GitFlow

### Criar uma Nova Feature

```bash
git checkout develop
git pull origin develop
git checkout -b feature/nome-da-feature

# Desenvolver e fazer commits
git commit -m "feat: descrição da feature"

# Finalizar feature
git checkout develop
git merge --no-ff feature/nome-da-feature
git branch -d feature/nome-da-feature
```

### Criar uma Release

```bash
git checkout develop
git checkout -b release/1.1.0

# Ajustar versão, atualizar CHANGELOG, etc.
git commit -m "chore: preparação para release 1.1.0"

# Finalizar release
git checkout main
git merge --no-ff release/1.1.0
git tag -a v1.1.0 -m "Release v1.1.0"
git checkout develop
git merge --no-ff release/1.1.0
git branch -d release/1.1.0
```

### Criar um Hotfix

```bash
git checkout main
git checkout -b hotfix/1.0.1

# Corrigir o problema
git commit -m "fix: descrição da correção"

# Finalizar hotfix
git checkout main
git merge --no-ff hotfix/1.0.1
git tag -a v1.0.1 -m "Hotfix v1.0.1"
git checkout develop
git merge --no-ff hotfix/1.0.1
git branch -d hotfix/1.0.1
```

## Convenções de Commit

Segue o padrão Conventional Commits:

- `feat:` Nova funcionalidade
- `fix:` Correção de bug
- `docs:` Documentação
- `style:` Formatação, falta ponto e vírgula, etc
- `refactor:` Refatoração de código
- `test:` Adição de testes
- `chore:` Manutenção (build, dependências, etc)
